# 01 本地启动
1. 执行 SQL：`db/sjc/V001 -> V002 -> V003 -> V004 -> V005`
2. 启动中间件：`docker compose -f deploy/docker-compose.yml up -d`
3. 启动后端：`cd spring && mvn -pl dev-admin spring-boot:run`
4. 启动前端：`cd web && npm run serve`

5. ClickHouse 初始化（可选）：`clickhouse-client --multiquery < db/ch/V001__inventory_history.sql`
6. 注意：`db/ch/*` 是 ClickHouse 语法，勿在 MySQL 中执行。


## 数据库账号密码填写位置（重要）
后端读取的是环境变量：`DB_USER` 和 `DB_PASS`（见 `spring/dev-admin/src/main/resources/application.yml`）。

可选方式（任选其一）：
1. **临时命令行传入（推荐）**
   - Linux/macOS：`cd spring && DB_USER=root DB_PASS=root mvn -pl dev-admin spring-boot:run`
   - Windows PowerShell：`$env:DB_USER="root"; $env:DB_PASS="root"; cd spring; mvn -pl dev-admin spring-boot:run`
2. **先导出环境变量再启动**
   - `export DB_USER=root`
   - `export DB_PASS=你的MySQL密码`
   - `cd spring && mvn -pl dev-admin spring-boot:run`
3. **直接改配置默认值（仅本机开发）**
   - 文件：`spring/dev-admin/src/main/resources/application.yml`
   - 把 `${DB_USER:root}` / `${DB_PASS:***}` 的默认值改成你本地真实账号密码。

如果你使用仓库里的 docker compose 默认 MySQL，账号通常是 `root/root`。
